磁盘存储及计算

核心定位：磁盘是典型的磁表面存储器，属于随机存取+顺序存取结合的设备，所有计算、概念都围绕「磁道、扇区、柱面」的物理结构展开，逻辑简单，无复杂考点。

一、磁盘存储 核心知识点（分模块，必背+考频标注+易错点，★=选择，★★=高频，★★★=大题必考）

✅ 模块1：磁盘的基本结构与概念（★★选择必考，填空，无计算，根基知识点，背熟不丢分）

所有概念成体系关联，选择题考「概念判断/对应关系」，无偏题，全部是送分点，按物理层级记忆即可。

1. 磁盘的物理结构（必考，磁盘计算的基础，所有公式的前提）

磁盘分为硬盘(HD) 和软盘，考题均以硬盘为核心，结构统一：

  • 盘片：磁盘的核心存储介质，一个磁盘有多个盘片，盘片有上下两个盘面（磁面），每个盘面都能存储数据；盘面数 = 盘片数 × 2。

  • 磁道：每个盘面被划分为多个同心圆的磁道，从外到内编号（0道为最外圈），同一盘面的所有磁道周长不同，存储容量相同（必考结论）。

  • 扇区：每个磁道被划分为多个扇区（扇段），扇区是磁盘的最小读写单位（★★★必考），每个扇区的存储容量固定（考题默认 $\boldsymbol{512Byte=0.5KB}$，必背！），扇区从0开始编号。

  • 柱面：所有盘面中，半径相同的磁道构成一个柱面，柱面数 = 单个盘面的磁道数（★★★必考）。

  • 磁头：每个盘面对应一个独立的读/写磁头，磁头数 = 盘面数，磁头可以沿半径方向移动，用于选择磁道。

2. 核心基础结论（★★选择必考，判断题高频，记牢直接秒杀）

  1. 磁盘的最小读写单位：扇区；CPU访问磁盘的最小单位：块（一个块对应一个/多个扇区）。

  2. 同一盘面，外圈磁道的位密度低、道密度高，内圈磁道的位密度高、道密度低，所有磁道的存储容量相同。

  3. 磁盘的地址格式（★★必考，填空）：$\boldsymbol{柱面号(磁道号) + 盘面号(磁头号) + 扇区号}$，寻址顺序：先找柱面→再找盘面→最后找扇区。

  4. 磁盘的存取特性：对磁道是随机存取（磁头移动到指定磁道），对扇区是顺序存取（磁道旋转到指定扇区）。

✅ 模块2：磁盘的两类核心容量计算（★★★大题必考计算，公式固定，送分题）

磁盘的容量分两种，考题必考区分+计算，公式简单，无灵活考点，背熟即可，是每年的送分计算，零失误。

通用约定：所有容量单位换算必背（考题必考）：$\boldsymbol{1Byte=8bit，1KB=1024B，1MB=1024KB，1GB=1024MB}$；扇区大小固定512B。

1. 格式化容量（★★★必考，实际可用容量）

  • 定义：磁盘格式化后的实际存储容量（划分了磁道、扇区，包含校验位、间隙，是用户真正能使用的容量）。

  • 核心公式（唯一，必背）：
    $\boldsymbol{格式化容量 = 盘面数 × 磁道数(柱面数) × 每道扇区数 × 每扇区字节数}$
    ✔ 变式：盘面数 = 磁头数，柱面数=磁道数 → 也可写为：容量 = 磁头数 × 柱面数 × 每道扇区数 × 512B。

2. 非格式化容量（★选择考点，物理最大容量）

  • 定义：磁盘未格式化的物理总容量（仅考虑磁道的位密度，无校验、无间隙，是磁盘的理论最大容量）。

  • 核心公式（唯一，必背）：
    $\boldsymbol{非格式化容量 = 盘面数 × 磁道数 × 磁道周长 × 位密度}$
    ✔ 补充：位密度的单位是 bit/mm，周长单位是mm，计算结果要转换为Byte（÷8）。

✔ 核心区别（★★选择必考）

格式化容量 < 非格式化容量，考题常考「磁盘的实际可用容量是？」→ 答案是格式化容量。

✅ 模块3：磁盘的核心性能指标+计算公式（★★★ 重中之重，大题压轴必考，分值最高，全计算，无例外）

本模块是磁盘的绝对核心考点，408每年必考磁盘性能指标计算大题（5~8分），所有公式必须背熟、记准，无灵活考点，所有计算都是套公式；同时也是选择题的高频考点，考「指标含义/公式判断」。核心前提：磁盘的一次完整读写过程，分为3个阶段，这是所有公式的根源，必背逻辑链：
$\boldsymbol{磁盘一次读写的总时间 = 寻道时间 + 旋转延迟时间 + 数据传输时间}$
✅ 所有性能指标，都是围绕这三个时间展开，记牢这个逻辑链，公式永远不会记混！

【前置必背：转速的单位换算（必考易错点，90%丢分在这里）】

磁盘的转速是核心已知条件，单位是 r/min（转/分钟），必须转换为 r/ms（转/毫秒） 或 ms/r（毫秒/转），公式唯一：
$\boldsymbol{磁盘旋转一周的时间 = 60000ms ÷ 转速(r/min)}$
例：转速为7200r/min → 旋转一周时间 = 60000/7200 ≈ 8.33ms；转速为5400r/min → 一周≈11.11ms。

1. 寻道时间 ($\boldsymbol{T_s}$)

  • 定义：磁头从当前磁道移动到目标磁道所需的时间，是磁盘读写的主要耗时（占比最大）。

  • 分类：

    • 平均寻道时间：磁头移动半程磁道数的平均时间，考题默认给出已知条件（如平均寻道时间为8ms），无需计算，直接用！
    • 最大寻道时间：磁头从最外圈到最内圈的时间，考题极少考。

  • 考点：寻道时间只和磁道的距离有关，和扇区、盘面无关（★★选择必考）。

2. 旋转延迟时间 ($\boldsymbol{T_r}$) 【必考计算，核心易错点】

  • 定义：磁头到达目标磁道后，目标扇区旋转到磁头下方所需的时间。

  • 核心公式（★★★必考，唯一，背熟）：
    $\boldsymbol{平均旋转延迟时间 = 磁盘旋转一周的时间 ÷ 2}$
    ✅ 致命易错点：必须除以2！因为扇区在磁道上均匀分布，平均旋转半周就能到达目标扇区，考题如果忘记÷2，整道题零分！
    ✔ 补充：最大旋转延迟时间 = 磁盘旋转一周的时间，考题极少考。

3. 数据传输时间 ($\boldsymbol{T_t}$) 【必考计算，核心】

  • 定义：磁头找到目标扇区后，读写扇区中数据的时间。

  • 核心公式（★★★必考，2个通用公式，全部背熟，考题二选一）：公式①（按扇区数计算，考题99%用这个，最常用）：
    $\boldsymbol{传输时间 = (读写的扇区数 ÷ 每道扇区数) × 磁盘旋转一周的时间}$
    公式②（按数据量计算）：
    $\boldsymbol{传输时间 = 读写的数据量 ÷ 磁盘的数据传输率}$

  • 考点：传输时间和磁道的扇区数、旋转速度有关，和寻道无关。

4. 平均存取时间 ($\boldsymbol{T_a}$) 【★★★大题必考，压轴核心，公式唯一】

  • 定义：磁盘完成一次平均读写操作的总时间，是磁盘的核心性能指标，必考计算！

  • 核心公式（唯一，必背，无例外，所有考题通用）：
    $\boldsymbol{平均存取时间 = 平均寻道时间 + 平均旋转延迟时间 + 平均传输时间}$
    ✅ 易错点补充：
    ① 选择题中，若题目忽略传输时间（如只问“磁盘的平均存取时间”，无数据量），公式简化为：$\boldsymbol{T_a = 平均寻道时间 + 平均旋转延迟时间}$；
    ② 大题中必须加传输时间，否则丢分，这是评分的采分点！

5. 磁盘带宽 ($\boldsymbol{B}$) 【★★计算考点，选择+大题】

  • 定义：磁盘单位时间内能够传输的数据量，单位是 Byte/s 或 MB/s，是磁盘的另一个核心性能指标。

  • 核心公式（2个，等价，全部背熟）：公式①：$\boldsymbol{带宽 = 每道扇区数 × 每扇区字节数 ÷ 旋转一周的时间}$
    公式②：$\boldsymbol{带宽 = 读写的数据量 ÷ 传输时间}$

  • 考点：带宽越大，磁盘的读写速度越快，带宽和转速成正比（转速越快，带宽越大）。

✅ 模块4：磁盘调度算法（★★★ 大题+选择必考，计算+概念双考点，分值高，无难点）

核心背景：磁盘的寻道时间是读写的主要耗时，磁盘调度算法的本质是：通过合理的磁头移动顺序，减少磁头的平均寻道时间，从而提升磁盘的整体性能（★★选择必考结论）。核心定义：磁盘调度算法也叫移臂调度算法，调度的是磁道（柱面），只改变磁头的移动顺序，不影响旋转延迟和传输时间（必考）。通用已知条件（所有算法通用）：给出当前磁头所在的磁道号 + 待访问的磁道号序列 + 磁头的移动方向（向内/向外，即磁道号减小/增大），要求：① 计算磁头移动的总磁道数；② 计算平均寻道长度（总磁道数÷请求数）。核心评分点：大题中，必须写出磁头的移动顺序，再计算总磁道数，只写结果会丢步骤分！

必考的4种调度算法（按考频排序，★★★必考前3种，第4种低频），全部有固定解题步骤，零失误

✔ 算法1：先来先服务（FCFS）【★★选择+计算，基础】

  1. 规则：按请求访问磁道的先后顺序，依次为各请求服务，无优先级，公平无歧视。

  2. 解题步骤：① 按请求顺序列出磁道移动路径；② 计算相邻磁道的差值之和（总磁道数）；③ 平均寻道长度=总磁道数/请求数。

  3. 特点：优点是算法简单、公平；缺点是磁头移动轨迹杂乱，平均寻道时间长，效率低，不适合负载大的场景。

  4. 核心结论（选择必考）：FCFS的平均寻道长度最长，效率最低。

✔ 算法2：最短寻道时间优先（SSTF）【★★★大题必考，高频核心】

  1. 规则：每次选择与当前磁头所在磁道距离最近的请求，优先服务，贪心策略。

  2. 解题步骤（万能模板）：① 确定当前磁道号；② 找出所有未服务的请求中，与当前磁道号差值最小的磁道；③ 移动磁头到该磁道，累加移动的磁道数；④ 以新磁道为当前磁道，重复②③，直到所有请求服务完毕。

  3. 特点：优点是平均寻道时间短，效率高；缺点是可能出现饥饿现象（距离远的请求长期得不到服务），无全局最优。

  4. 核心结论（选择必考）：SSTF的平均寻道长度远小于FCFS，但存在饥饿。

✔ 算法3：扫描算法（SCAN）【★★★大题必考，压轴核心，也叫「电梯调度算法」】

  1. 规则：磁头沿一个固定方向（向内/向外）移动，依次服务所有经过的磁道请求，直到到达该方向的最末端磁道；然后反转方向，沿反方向移动，继续服务剩余请求，完全模拟电梯的运行规则（必考名称）。

  2. 解题步骤（万能模板，大题采分点）：① 确定当前磁道号+磁头移动方向；② 按移动方向，从小到大/从大到小列出所有未服务的磁道；③ 依次移动磁头，累加移动的磁道数；④ 到达末端后反转方向，处理剩余磁道，累加磁道数；⑤ 计算总磁道数和平均寻道长度。

  3. 特点：优点是无饥饿现象，平均寻道时间短，效率高，是实际中最常用的算法；缺点是对两端的磁道请求服务延迟稍大。

  4. 核心结论（选择必考）：SCAN算法无饥饿、全局最优，效率优于SSTF和FCFS。

✔ 算法4：循环扫描算法（C-SCAN）【★选择考点，低频计算】

  1. 规则：是SCAN的改进版，磁头沿一个固定方向移动，服务所有经过的磁道请求，到达末端后，直接跳回起始磁道（最外圈/最内圈），不反转方向，继续沿原方向服务剩余请求。

  2. 特点：优点是所有磁道的请求被服务的平均延迟相同；缺点是磁头移动的总磁道数比SCAN多，效率略低。

✔ 4种算法的核心对比（★★选择必考，表格速记，一目了然）

调度算法 核心规则 平均寻道长度 饥饿现象 效率 实际应用
FCFS 按请求顺序 最长 无 最低 轻负载
SSTF 最短距离优先 较短 有 较高 中负载
SCAN 电梯调度，双向移动 短 无 最高 主流，重负载
C-SCAN 单向移动，跳回起点 中等 无 较高 对延迟敏感的场景

✅ 模块5：磁盘的易错点+避坑指南（选择题/计算题高频丢分点，必看，避坑=提分）

这部分是磁盘模块唯一的丢分点，全部是细节问题，记牢这些规则，能避免99%的错误，重中之重！

  1. 扇区大小固定为512B：考题中无论是否说明，都按512B计算，无需额外推导；

  2. 旋转延迟必须÷2：平均旋转延迟是半周时间，大题忘记÷2直接零分；

  3. 存取时间的公式：大题必须加传输时间，选择题可忽略，看题目要求；

  4. 转速的单位换算：转速是r/min，旋转一周的时间是ms，必须用60000÷转速，不是60÷转速；

  5. 磁盘调度调度的是磁道：不是扇区、不是盘面，只影响寻道时间，不影响旋转和传输时间；

  6. 格式化容量的公式：盘面数=磁头数，柱面数=磁道数，不要漏乘盘面数；

  7. 单位换算：1KB=1024B，1MB=1024KB，不要和1000混淆（计组中所有存储单位都是2的幂次）。

二、磁盘存储 三大核心题型（解题模板+例题+答案，408必考，全覆盖所有考点，无偏题）

题型标注：★★★=大题必考（5~8分），★★=选择必考（2分），★=低频填空；所有题目均为408真题改编，贴合考试难度，解题步骤固定，模板化答题零失误。

✅ 题型1：磁盘容量+性能指标计算题（★★★ 大题必考，核心计算，分值最高，送分题）

题型特征

已知：磁盘的盘面数、磁道数、每道扇区数、转速、平均寻道时间，要求：① 计算格式化容量；② 计算旋转一周的时间、平均旋转延迟；③ 计算平均存取时间；④ 计算磁盘带宽。

万能解题步骤（死记，按顺序写，采分点齐全）

  1. 计算格式化容量：套公式 → 盘面数×磁道数×每道扇区数×512B；

  2. 计算旋转一周的时间：60000ms ÷ 转速；

  3. 计算平均旋转延迟：旋转一周的时间 ÷ 2；

  4. 计算传输时间：(读写扇区数/每道扇区数) × 旋转一周的时间；

  5. 计算平均存取时间：平均寻道时间 + 平均旋转延迟 + 传输时间；

  6. 计算带宽：(每道扇区数×512B) ÷ 旋转一周的时间。

经典例题（408真题高频，必考大题，8分）

已知某磁盘有4个盘片，转速为7200r/min，平均寻道时间为8ms，每个盘面有2048个磁道，每个磁道有128个扇区，扇区大小为512B。求：
(1) 该磁盘的格式化容量（单位：GB）；
(2) 磁盘旋转一周的时间和平均旋转延迟时间；
(3) 读取一个扇区的平均存取时间；
(4) 该磁盘的数据传输率（单位：MB/s）。

标准答案+步骤

(1) 盘面数=4×2=8；格式化容量=8×2048×128×512B = 8×2048×128×0.5KB = 8×2048×64KB = 8192MB = 8GB；
(2) 旋转一周时间=60000/7200 ≈8.33ms；平均旋转延迟=8.33/2≈4.17ms；
(3) 读取1个扇区，传输时间=(1/128)×8.33≈0.065ms；平均存取时间=8+4.17+0.065≈12.235ms；
(4) 传输率=(128×512B)/8.33ms = 65536B/8.33ms ≈7.87MB/s。

✅ 题型2：磁盘调度算法计算题（★★★ 大题必考，核心计算，分值高）

题型特征

已知：当前磁头所在磁道号、待访问的磁道序列、磁头移动方向，要求：① 写出磁头的移动顺序；② 计算磁头移动的总磁道数；③ 计算平均寻道长度。

万能解题步骤（核心模板，所有算法通用，大题采分点）

  1. 明确调度算法+当前磁道+移动方向；

  2. 按算法规则，列出磁头的完整移动顺序（大题必写，采分点）；

  3. 计算相邻磁道的差值，累加得到总磁道数；

  4. 平均寻道长度 = 总磁道数 ÷ 待访问的请求数。

经典例题（408真题改编，压轴大题，7分）

已知磁盘的磁道号为0~199，当前磁头位于100号磁道，磁头正在向磁道号增大的方向移动，待访问的磁道序列为：55、90、150、30、180、120。分别用：① SSTF算法；② SCAN算法，计算磁头移动的总磁道数和平均寻道长度。

标准答案+步骤

① SSTF算法（最短距离优先）：移动顺序：100 →90 →120 →150 →180 →55 →30；总磁道数：|100-90|+|90-120|+|120-150|+|150-180|+|180-55|+|55-30| =10+30+30+30+125+25 = 250；平均寻道长度=250/6≈41.67。

② SCAN算法（电梯调度，向外移动）：移动顺序：100 →120 →150 →180 →90 →55 →30；总磁道数：20+30+30+90+35+25 = 230；平均寻道长度=230/6≈38.33。

经典简答真题（必考，6分）

简述磁盘调度算法的设计目的，并对比FCFS、SSTF、SCAN三种算法的优缺点。

标准答案（采分点清晰，答出要点即可满分）

  1. 设计目的：减少磁头的平均寻道时间，提升磁盘的读写效率。

  2. 优缺点：

    • FCFS：优点是算法简单、公平；缺点是平均寻道时间长，效率低。
    • SSTF：优点是平均寻道时间短，效率高；缺点是存在饥饿现象。
    • SCAN：优点是无饥饿现象，平均寻道时间短，效率最高；缺点是两端请求的服务延迟稍大。

  2. 磁盘调度算法：① FCFS：按请求顺序，公平但效率低；② SSTF：最短距离优先，效率高但有饥饿；③ SCAN：电梯调度，双向移动，无饥饿、效率最高；④ C-SCAN：单向移动，跳回起点，无饥饿、延迟均匀。

  3. 原因：寻道时间占磁盘总存取时间的比例最大，是主要耗时。解决方式：通过合理的磁头移动顺序，减少磁头的平均移动距离，从而降低平均寻道时间。

★ 容量公式

  1. 格式化容量 = 盘面数 × 磁道数 × 每道扇区数 × 512B

  2. 非格式化容量 = 盘面数 × 磁道数 × 周长 × 位密度 ÷8

★ 性能指标公式

  1. 旋转一周的时间 = 60000ms ÷ 转速(r/min)

  2. 平均旋转延迟 = 旋转一周的时间 ÷ 2

  3. 传输时间 = (读写扇区数/每道扇区数) × 旋转一周的时间

  4. 平均存取时间 = 平均寻道时间 + 平均旋转延迟 + 传输时间

  5. 磁盘带宽 = (每道扇区数×512B) ÷ 旋转一周的时间

★ 调度算法公式

  1. 总磁道数 = Σ|当前磁道 - 下一个磁道|

  2. 平均寻道长度 = 总磁道数 ÷ 待访问的请求数
