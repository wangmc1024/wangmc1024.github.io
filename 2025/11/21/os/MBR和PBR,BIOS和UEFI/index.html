MBR和PBR,BIOS和UEFI

MBR、PBR、BIOS与UEFI详解

一、MBR(主引导记录)与PBR(分区引导记录)

1. MBR (Master Boot Record)

  • 位置：硬盘第一个扇区(LBA 0)，512字节
  • 结构：
    • 446字节：引导代码(负责加载操作系统)
    • 64字节：分区表(DPT，最多记录4个主分区)
    • 2字节：魔数(0x55AA，用于验证有效性)
  • 作用：
    • 系统启动时首个被读取的扇区
    • 检查分区表，定位活动分区
    • 将控制权交给活动分区的PBR
  • 限制：最大支持2TB硬盘，最多4个主分区

2. PBR (Partition Boot Record)

  • 位置：每个分区的第一个扇区
  • 特点：
    • 由高级格式化(FORMAT)写入
    • 与文件系统相关(如FAT32、NTFS不同)
  • 作用：
    • 加载分区对应的文件系统驱动
    • 启动分区上的操作系统引导程序
    • 对Windows：XP及更早版本引导NTLDR；Vista及以上引导bootmgr

3. MBR与PBR的关系

  • MBR是硬盘全局引导记录，PBR是分区局部引导记录
  • 启动流程：BIOS→MBR→(选择活动分区)→PBR→操作系统

二、BIOS与UEFI

1. BIOS (Basic Input/Output System)

  • 定义：主板ROM中的固件，计算机启动后首个运行的程序
  • 工作模式：16位实模式，依赖中断机制
  • 启动流程：
    1. 执行POST(开机自检)
    2. 读取MBR到内存并执行
    3. MBR引导代码加载PBR
    4. PBR启动操作系统
  • 局限：
    • 仅支持MBR分区，最大2TB硬盘
    • 仅文本界面，操作繁琐
    • 启动较慢，扩展性差

2. UEFI (Unified Extensible Firmware Interface)

  • 定义：BIOS的现代替代品，2007年由Intel推出
  • 核心优势：
    • 支持32/64位执行环境
    • 图形化界面，支持鼠标操作
    • 模块化设计，可扩展性强
    • 并行初始化硬件，启动速度提升30%+
  • 启动方式：
    • 直接读取EFI系统分区(ESP)中的.efi文件
    • 支持GPT分区表(最大9.4ZB，128个分区)
    • 内置安全启动(Secure Boot)，防止未授权系统加载

3. BIOS与UEFI对比

特性 BIOS UEFI
磁盘支持 MBR(≤2TB,4主分区) GPT(≤9.4ZB,128分区)
启动方式 MBR→PBR→OS 直接加载ESP分区的.efi文件
界面 纯文本，键盘操作 图形界面，支持鼠标
安全性 无安全启动 支持Secure Boot
网络功能 无 部分支持网络启动、远程维护
适用场景 老旧系统，小硬盘 Windows 11等现代系统，大容量存储

三、系统启动全流程

BIOS+MBR启动流程：

  1. 开机→BIOS自检
  2. BIOS读取硬盘MBR
  3. MBR检查分区表，找到活动分区
  4. MBR将控制权交给活动分区的PBR
  5. PBR加载分区文件系统和引导程序(如Windows的bootmgr)
  6. 引导程序加载操作系统内核

UEFI+GPT启动流程：

  1. 开机→UEFI初始化
  2. UEFI直接访问ESP分区( FAT32格式)
  3. 读取并执行分区中的.efi引导文件
  4. 引导程序加载操作系统

四、选择建议

  • 使用BIOS+MBR：
    • 硬盘<2TB且分区数≤4
    • 兼容老旧系统
    • 主板不支持UEFI
  • 使用UEFI+GPT：
    • 硬盘>2TB
    • 需要更多分区(最多128个)
    • Windows 11等现代系统(强制要求)
    • 追求更快启动速度和更好安全性

总结：MBR和PBR是硬盘引导的”基础设施”，BIOS和UEFI则是决定如何使用这些设施的”交通规则”。随着硬件发展，UEFI+GPT已成为主流，提供更大存储支持、更快启动和更强安全性，而BIOS+MBR则逐渐退居兼容模式。
