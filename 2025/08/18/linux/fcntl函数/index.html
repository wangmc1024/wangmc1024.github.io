使用场景和相关用法

fcntl函数

一、fcntl 函数基础

  • 返回值：成功时返回与 cmd 相关的值（如标志、文件描述符），失败返回 -1 并设置 errno。

  • 核心作用：通过 cmd 命令对 fd（文件描述符）执行控制操作，涵盖文件状态、I/O 模式、锁机制等。

二、与 O_NONBLOCK 相关的操作（文件状态标志）

O_NONBLOCK 是 非阻塞模式标志，用于设置文件描述符的 I/O 行为：

  • 阻塞模式（默认）：当执行 read/write 等操作时，若暂时无法完成（如无数据可读、缓冲区满），进程会进入睡眠等待。

  • 非阻塞模式（O_NONBLOCK）：操作无法立即完成时，函数会立即返回 -1 并设置 errno 为 EAGAIN 或 EWOULDBLOCK，不阻塞进程。

相关 cmd 命令：

  1. F_GETFL：获取当前文件状态标志（包含是否设置 O_NONBLOCK）

  2. F_SETFL：设置文件状态标志（用于开启 / 关闭 O_NONBLOCK）

适用场景：

  • 网络套接字（socket）：避免 recv/send 阻塞导致进程卡死。

  • 管道（pipe）、FIFO：处理读写端不同步的情况。

  • 设备文件（如终端、串口）：实时响应 I/O 状态变化。

三、文件锁操作（进程间同步）

fcntl 支持对文件的部分或全部内容加锁，避免多进程竞争。相关 cmd 命令需配合 struct flock 结构体使用：

  1. F_GETLK：检查锁冲突（获取当前阻塞当前锁请求的锁信息）

  2. F_SETLK：设置或释放锁（非阻塞模式，失败立即返回）

  3. F_SETLKW：设置或释放锁（阻塞模式，若无法获取则等待）

四、文件描述符管理

  1. F_DUPFD：复制文件描述符（类似 dup）

  2. F_DUPFD_CLOEXEC：复制描述符并设置 FD_CLOEXEC 标志

  3. F_GETFD/F_SETFD：获取 / 设置文件描述符标志（仅 FD_CLOEXEC 有效）

五、异步 I/O 与信号通知

  1. F_SETOWN：设置接收 SIGIO 和 SIGURG 信号的进程 / 进程组

  2. F_GETOWN：获取当前接收信号的进程 / 进程组 ID

  3. F_SETFL 配合 O_ASYNC：开启异步 I/O 通知（数据可读 / 可写时触发 SIGIO）

六、其他系统相关命令

  • F_GETPIPE_SZ/F_SETPIPE_SZ：获取 / 设置管道（pipe）的缓冲区大小

  • F_ADD_SEALS/F_GET_SEALS：为文件添加 / 查询 “密封” 标志（限制修改）

  • F_GETLEASE/F_SETLEASE：获取 / 设置文件租约（协调进程间文件访问）

fcntl 是一个功能强大的多用途函数，核心能力包括：

  1. 控制 I/O 模式（如 O_NONBLOCK 非阻塞）；

  2. 进程间文件锁（读锁、写锁）；

  3. 文件描述符管理（复制、标志设置）；

  4. 异步 I/O 信号通知；

  5. 系统特定扩展功能（管道大小、文件密封等）。

使用时需注意：不同 Unix/Linux 系统对命令的支持可能有差异，建议通过 man 2 fcntl 查看本地系统的详细文档。非阻塞模式（O_NONBLOCK）常与 I/O 多路复用（select/poll/epoll）结合，是高性能网络编程的基础。
