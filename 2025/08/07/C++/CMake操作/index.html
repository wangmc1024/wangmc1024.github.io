关于CMake的基础概念和CMake项目构建流程

一、CMake 基础语法

1. 基础语法规则

  • 命令格式：不区分大小写（惯例用大写），格式为 COMMAND(参数1 参数2 ...)，参数含空格需用双引号包裹。
    例：PROJECT(MyApp) 与 project(myapp) 等效。

  • 变量：

    • 定义：set(变量名 值)，如 set(SRC main.cpp)。
    • 引用：${变量名}，如 add_executable(app ${SRC})。
    • 类型：
      • 普通变量：当前作用域有效（函数/目录内）。
      • 缓存变量：set(变量名 值 CACHE 类型 "描述")，存入 CMakeCache.txt 跨构建保留（如 CMAKE_BUILD_TYPE）。
      • 环境变量：$ENV{变量名} 引用，set(ENV{变量名} 新值) 修改。
    • 预定义核心变量：
      变量名 含义
      PROJECT_SOURCE_DIR 项目根目录（根 CMakeLists.txt 路径）
      CMAKE_BINARY_DIR 构建目录（执行 cmake 命令的目录）
      CMAKE_CXX_STANDARD C++ 标准版本（如 17、20）

  • 列表（List）：空格分隔的字符串集合，如 set(LIBS math utils)。
    常用操作：list(APPEND 列表 元素)（追加）、list(LENGTH 列表 变量)（长度）。

  • 注释：单行用 #，无多行注释（需每行加 #）。

二、核心构建命令

1. 项目配置基础

  • cmake_minimum_required(VERSION 版本)：指定最低 CMake 版本（必须在 project 前），如 cmake_minimum_required(VERSION 3.10)。

  • project(项目名 [LANGUAGES C CXX ...])：定义项目名称及支持语言（默认 C/C++），自动生成 PROJECT_NAME、PROJECT_VERSION 等变量。

2. 目标（Target）构建

目标是 CMake 核心，代表可执行文件、库等产物。

  • 可执行文件：add_executable(目标名 源文件...)
    例：add_executable(app main.cpp src/func.cpp)。

  • 库文件：add_library(目标名 [STATIC|SHARED|MODULE] 源文件...)

    • STATIC：静态库（.a/.lib，编译时链接）。
    • SHARED：动态库（.so/.dll，运行时加载）。
    • MODULE：模块库（非链接用，运行时动态加载）。
      例：add_library(math STATIC src/math.cpp)。

3. 目标属性与依赖配置

现代 CMake（3.0+）推荐"目标导向"配置，替代全局命令（如 include_directories），明确作用域：

  • target_include_directories(目标 作用域 目录...)：指定头文件目录

    • PUBLIC：当前目标及依赖它的目标可见。
    • PRIVATE：仅当前目标可见。
    • INTERFACE：仅依赖它的目标可见。
      例：target_include_directories(app PUBLIC include)。

  • target_link_libraries(目标 依赖...)：链接库（自定义库或系统库）
    例：target_link_libraries(app PUBLIC math pthread)（链接自定义库 math 和系统线程库）。

  • target_compile_features(目标 作用域 cxx_std_17)：指定 C++ 标准（自动配置编译选项）。

  • target_compile_options(目标 作用域 -Wall -O2)：添加编译选项（警告、优化等）。
