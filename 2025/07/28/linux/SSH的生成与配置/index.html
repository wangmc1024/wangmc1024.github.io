常见问题解决

  1. 权限错误： 若提示 “Permission denied”，检查服务器~/.ssh和authorized_keys的权限是否为 700 和 600。
  2. 端口问题： 服务器 SSH 端口非默认时，连接需指定端口：ssh -p 2222 用户名@IP，或在config中配置Port。
  3. 密钥未生效： 重启服务器 SSH 服务：sudo systemctl restart sshd（Linux）。
