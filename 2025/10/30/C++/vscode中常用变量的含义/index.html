vscode中常用变量的含义

发表于 2025-10-30 分类于 编程 阅读次数：
本文字数： 1.8k 阅读时长 ≈ 2 分钟
变量参数 核心含义 高频使用场景
${workspaceFolder} 当前工作区根目录的绝对路径（首个打开文件夹的完整路径） 配置构建/运行的根目录、指定输入输出路径
${file} 当前打开文件的绝对路径（含完整路径、文件名+扩展名） 调试单个文件、指定待执行文件路径
${fileBasenameNoExtension} 当前文件的名称（不含扩展名） 生成对应输出文件（如 .exe 或日志文件）
${fileDirname} 当前文件所在文件夹的绝对路径 引用同目录下的依赖文件、输出到文件所在目录
${relativeFile} 当前文件相对于工作区根目录的相对路径 多文件项目中指定相对路径引用
${workspaceFolderBasename} 工作区根目录的文件夹名称（不含路径） 命名构建产物、区分多工作区项目
${fileBasename} 当前文件的完整名称（含扩展名） 直接引用当前文件全名（如读取配置文件）
${fileExtname} 当前文件的扩展名（带开头的点，如 .cpp） 根据文件类型动态配置执行逻辑
${fileWorkspaceFolder} 当前文件所在的工作区根目录（适配多根工作区） 多项目并存时精准定位文件所属工作区
${cwd} 命令执行时的工作目录（默认与 ${workspaceFolder} 一致） 配置终端执行目录、指定程序运行目录
${env:XXX} 访问系统环境变量（XXX 为具体环境变量名，如 ${env:PATH}） 引用系统路径、环境变量配置（如 Python 路径）
${config:XXX} 访问用户/工作区配置项（XXX 为具体配置名，如 ${config:python.pythonPath}） 读取 VS Code 自定义配置参数
${execPath} VS Code 可执行程序的完整路径 调用 VS Code 相关工具、插件依赖路径配置
${defaultBuildTask} 默认的构建任务名称 关联调试与构建任务、自动执行默认构建
${lineNumber} 当前光标所在的行号 调试时定位断点行、日志输出行号标记
${selectedText} 当前编辑器中选中的文本内容 动态替换配置参数、传递选中内容到命令

基于固定场景：

  • 工作区根目录（打开的文件夹）：D:\MyCode\PythonProject

  • 当前打开文件（编辑中）：D:\MyCode\PythonProject\src\utils\data_process.py

以下是每个变量参数对应的具体值：

变量参数 对应具体值（基于上述场景）
${workspaceFolder} D:\MyCode\PythonProject
${file} D:\MyCode\PythonProject\src\utils\data_process.py
${fileBasenameNoExtension} data_process
${fileDirname} D:\MyCode\PythonProject\src\utils
${relativeFile} src\utils\data_process.py
${workspaceFolderBasename} PythonProject
${fileBasename} data_process.py
${fileExtname} .py
${fileWorkspaceFolder} D:\MyCode\PythonProject
${cwd} D:\MyCode\PythonProject（默认值）
${env:PATH} C:\Windows\system32;D:\Python311\;C:\Program Files\...（你的系统PATH路径）
${config:python.pythonPath} D:\Python311\python.exe（你在VS Code中配置的Python路径）
${execPath} C:\Users\你的用户名\AppData\Local\Programs\Microsoft VS Code\Code.exe
${defaultBuildTask} C/C++: gcc 编译当前文件（假设你配置的默认构建任务名）
${lineNumber} 30（假设光标当前停在第30行）
${selectedText} df = pd.read_csv("data.csv")（假设你选中的代码行）
